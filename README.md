## 1. Зачем используется Kanban-доска в проекте

Kanban-доска помогает:
- Визуализировать весь процесс разработки
- Отслеживать прогресс выполнения задач
- Распределять нагрузку между участниками команды

## 2. Значение колонок
Backlog - все задачи проекта
To Do - задачи, готовые к работе (определен приоритет, есть исполнитель)
In Progress - задачи, которые сейчас в работе
Review- задачи, требующие проверки (код-ревью, тестирование, согласование)
Done - завершенные задачи

## 3. Как создаётся новая задача

1. Перейдите на вкладку Issues в репозитории
2. Нажмите New issue
3. Заполните:
   - Title: название задачи
   - Description: подробное описание, user stories
4. В правой панели:
   - Выберите Projects: "Service Exchange — Development Board"
   - Установите Status: Backlog
   - Выберите Role (Backend/Frontend/Design/QA/Analytics)
   - Установите Priority (High/Medium/Low)
   - Назначьте Assignee\

## 4. Путь задачи от Backlog до Done
- Backlog → To Do: задача приоритизирована, назначен исполнитель
- To Do → In Progress: разработчик начал работу
- In Progress → Review: задача выполнена, требует проверки
- Review → Done: проверка пройдена, задача завершена
- Review → In Progress: обнаружены ошибки, требуется доработка

## 5. Распределение задач между ролями
Задачи распределяются с помощью поля Role:

Backend - разработка API, серверной логики, работа с БД
Frontend - разработка пользовательского интерфейса
Design - создание макетов, UI/UX дизайн
QA - тестирование, написание тест-кейсов
Analytics - составление ТЗ, анализ требований

## 6. Как отслеживать прогресс проекта

- Внимание на задачи с High приоритетом
- Скорость движения задач по доске
- Количество задач в Review
- Соотношение задач по ролям
